Fixing Git Mistakes

  1. Rename last commit
    • let's commit: "git commit -m "Add index.htmll"
    • Now we want to change the commit message: git commit --amend -m "Add index.html"
    • Check last commit log: "git log --oneline"

  2. Add new changes to last commit
    • Make changes and add untracked file : "git add -A"
    • Add new changes to last commit: git commit --amend -m "Adding index.html and app.js"
    • Check the last commit: "git log --oneline"

  3. Git remove file from staging before commiting
    • Let's add new file: "touch lib.js"
    • Add this file to staging area: "git add lib.js"
    • Check the status of file "git status" and check the head position by "git log --oneline"
      • Ex: 6cdfc0e (HEAD -> master) Adding index.html and app.js
        f86d914 (origin/master, origin/HEAD) Initial commi"
    • Now reset head : "git reset HEAD lib.js"
    • Check the status and remove the file.

  4. Git remove changes from a commit before pushing
    • Let's add new file app.js and add some lines
    • Add changes to stage area and do a commit: "git add -A" && "git commit -m "Adds a desc to index""
    • Now check the diff from origin : "git diff origin/master HEAD"
    • Check git log : "git log --oneline"
      • It should show something like this:
        • 93a6379 (HEAD -> master) Adds a desc to index
        • cbbc0cb Adding index.html and app.js
        • f86d914 (origin/master, origin/HEAD) Initial commit
    • Now we have two ways to remove changes from a commit:
      1. Reset head by one commit : "git reset HEAD~1"
      2. Or Reset head by commit ID : "git reset 93a6379"

  5. Git different reset options - soft/mixed/hard
    Soft Reset - It undo the changes from the given commit and update the state of changed files into staging area. Such as:
    • Let's add a file and commit : "git add myScript.js" && "git commit -m 'take 1'"
    • Check log status : "git log --oneline"
      • 502381c (HEAD -> master) take 1
        cbbc0cb Adding index.html and app.js
        f86d914 (origin/master, origin/HEAD) Initial commit
    • Now let's do soft reset : "git reset --soft HEAD~1"
    • Again chek git log status: "git log --oneline"
      • 502381c (HEAD -> master) take 1 cbbc0cb Adding index.html and app.js
        f86d914 (origin/master, origin/HEAD) Initial commit
    • Now undo files are in staging area. Check status: "git status"
      • On branch master Your branch is ahead of 'origin/master' by 1 commit. (use "git push" to publish your local commits)
        Changes to be committed:
        (use "git restore --staged ..." to unstage) new file: myScript.js

    Mixed Reset - Reset changes fis hardorm commit and unlike soft reset, It also remove commit changes from staging area:
    • git reset --mixed HEAD~1

    Hard Reset - This option we rarely use in scenario where we want to completely remove commit's changes from the working directory.
    ( This option is not advisable because undo of this command is complicated: How to undo git hard reset)
